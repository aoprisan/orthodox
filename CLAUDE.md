## Project

A Christian Orthodox liturgical calendar SPA: feast days, saints, fasting indicators, and full Romanian "Lives of the Saints" hagiography. Switches between Revised Julian ("new") and Julian ("old") calendars and between English/Romanian. Built with Vite + React + TypeScript; ships to the web via GitHub Pages and as native iOS/Android apps via Capacitor.

## Architecture

State lives in `src/App.tsx` via plain `useState` â€” no Redux/Context. The four state values are `selected` (Date), `kind` (`'new' | 'old'`), `lang` (`'en' | 'ro'`), and the visible month. `kind` and `lang` persist to `localStorage`; `selected` is mirrored to a `#/YYYY/MM/DD` URL hash (`parseHash`/`writeHash` + a `hashchange` listener). Render tree: `Header` (calendar/lang toggles, nav) â†’ `TodayCard` â†’ `MonthGrid`â†’`DayCell` (the grid) and `DayDetail` (the selected-day panel, which lazy-loads hagiography via `SaintLifeSection`).

All liturgical logic is pure functions in `src/lib/`, keyed off `(date, kind)`:
- `pascha.ts` â€” Orthodox Easter (Meeus Julian algorithm); the anchor for everything moveable.
- `moveableFeasts.ts` â€” Triodion/Lent/Pentecost cycle computed from Pascha. **Moveable feasts use the same civil (Gregorian) date for both `kind`s** because Pascha is a single liturgical event.
- `julian.ts` â€” Gregorianâ†”Julian conversions. **Fixed-date data is looked up on the Julian date when `kind === 'old'`** (this is why Nativity lands on Dec 25 in "new" but Jan 7 in "old").
- `feasts.ts` (`entryForDate`) â€” merges fixed feasts + Romanian saints + extended synaxarion + moveable feasts into one `DayEntry`, deduplicating saints by normalized name.
- `fasting.ts` (`fastFor`), `services.ts` (`servicesFor`) â€” derive fast level/reason and a *typical* (not canonical) parish schedule.

### Data model and the bilingual convention

The central concept is the `Localized` type in `src/types.ts`: a name is either a plain `string` (English-only) or `{ en, ro }`. The `loc()` helper in `src/i18n/loc.ts` resolves it. **Romanian view deliberately hides untranslated English-only entries** so no English leaks into RO mode â€” keep this in mind when adding saints.

Saints come in two tiers, merged at lookup in `feasts.ts`:
- **Principal** â€” hand-curated, bilingual: `src/data/fixedFeasts.ts` (full year, `"MM-DD"` keys) and `src/data/romanianSaints.ts` (modern Romanian/Synod canonizations not on calendar-ortodox.ro).
- **Secondary** â€” auto-generated, lower visual weight: `src/data/extendedSynaxarion.ts` (regenerated by the build scripts; do not hand-edit).

Add a new fixed feast/saint by editing `src/data/fixedFeasts.ts` with a `{ en, ro }` name â€” no rebuild needed (HMR). Great Feasts belong in `fixedFeasts.ts`, **not** the extended synaxarion (the scrapers carry feast-word skip lists to keep them out).

### Scraper fragility (calendar-ortodox.ro)

Parsing lives in `scripts/lib/parseCaoLives.mjs` (prose) and `scripts/lib/parseCaoNames.mjs` (short names). The source is legacy HTML with several traps that have caused regressions â€” keep them in mind when touching parsers:
- **Mixed encoding** â€” pages are a mix of UTF-8 and iso-8859-2, sniffed from `<meta>`; diacritics arrive as named/numeric HTML entities; Ĺź/ĹŁ (cedilla) are normalized to Č™/Č› (comma-below).
- **`sec.` abbreviation** â€” sentence splitting must not break on the period in "(sec. V)"; `parseCaoNames.mjs` uses a `(?<!\bsec)\.` guard.
- **Unclosed `<P>` tags** â€” sections must be split on opening `<p>`, not strict `<p>â€¦</p>` pairs, or trailing paragraphs are dropped.
